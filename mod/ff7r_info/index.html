はじめに

ついにPC版FF7Rが発売されました！
多くの方にとってはどうでもいいニュースだと思いますが、PS版を買わずにPC版を待ち続けた僕にとっては今年一の大ニュースです。
発売後すぐにソフトを購入し、MOD作成のために情報を集めています。

このページでは、FF7RのMOD作成について僕が集めた情報を記載します。

MOD作成の大まかな流れ

FF7RはUE4で作られているため、他のUE4製ゲームと似たような手順でMODが作れます。
以下のようなMOD作成ガイドを参考にしましょう。

『聖剣伝説3 TRIALS of MANA』UE4用MOD作成ガイド [21/10/22 キーについて追記]｜HAL’s Blog

(ただし、後述する理由により、同じ手順ではMODを作ることができないので注意してください。)

独自仕様のUE4による弊害

さきほど紹介したMOD作成ガイドをなぞるだけではFF7RのMODをつくることができません。
なぜなら、FF7Rで使用されているUE4は公式で配布されているどのバージョンとも異なるからです。

UE4の大規模カスタマイズが支えた"懐かしくも新しい『FFVII』"～『FINAL FANTASY VII REMAKE』（1）開発体制&キャラクター制作 | 特集 | CGWORLD.jp
↑の記事によると、

「4.18の時点でバージョンをロック。それ以降のアップデートに際しては、最適化系の機能を部分的にマージしていった。」

とのこと。

つまり、バージョンは4.18になってるけど、それ以降のバージョンの機能も使っている、 ということです。

その他にも、補助骨システムや描画プロセスなど、本来のUE4には存在しない独自仕様が多々あります。
あまりの改造っぷりに、MOD界隈では「UE4フランケンシュタインver」と形容されるほどです。

これにより、
「MOD用のツールは公式のバージョン4.18を想定して動作するが、想定外の機能が使われていてエラーが起こる。」
という状況が発生します。
QuickBMSやUE Viewerなど、各種ツールはFF7R専用にカスタマイズされたバージョンを使う必要があるのです。

OpenFF7R

OpenFF7RはFF7RのMODコミュニティです。
FF7RのMODに関する情報は全てOpenFF7Rに集約されています。
OpenFF7Rへ参加せずにMODを作ることは不可能と言っても過言ではありません。

1.Discordサーバーへの参加方法

OpenFF7Rに貼られているリンクからDiscordサーバーに参加できます。
ただし、MOD関連のチャンネルにアクセスするためには、ボットに認証してもらう必要があります。
"#reaction-roles"のチャンネルに投稿されているボットのメッセージに対して"⚔"(2本の剣のマーク)をクリックすると、他チャンネルへのアクセスが許可されます。

2.Modderチャンネルへの参加方法

Discordサーバーには"#modders-only"というMOD作成者専用のチャンネルが存在しており、MOD作成に関する議論は主にこの専用チャンネルで行われています。
以下の手順でModderチャンネルにアクセスできるようになります。

  1. UE4製ゲームのMODを作る
  2. 自分が作成したMODの公開ページへのリンクを"#role-request"に投稿する
  3. モデレータが投稿を見て、MODDERの役職を付与する
役職がMODDERに変更されると、"#role-request"に投稿した自分のメッセージが削除され、"#modders-only"にアクセスできるようになっているはずです。
なお、"#role-request"に投稿する際は、モデレータの方にDMを送ったり、投稿をピン留めしたりしてはいけません。

(2021/01/02追記)
OpenFF7Rの管理人から「テクスチャを入れ替えるだけのMODではMODDER認定しない」との通達が出されました。
便利なツールが出回り、テクスチャMODが量産されている現状を加味したものと思われます。

3.Discordサーバーの活用方法

Dicordサーバー内の有用なチャンネルをいくつか紹介します。

  • #resources: ツールを投稿するチャンネルです。必要なツールの殆どはここで手に入ります。
  • #modding-tutorials: チュートリアルを投稿するチャンネルです。MOD作成に関する情報はここで手に入ります。
  • #modding-help: 質問用のチャンネルです。MOD作成に躓いたらここで質問しましょう。

各種ツール

MODの作成に必要なツールを紹介していきます。
ただし、Blenderのような一般的なツールの紹介は省き、MOD専用のツールのみ紹介します。
また、一部のツールはリンクを貼りません。OpenFF7R内で探してください。

1.必須ツール

まず初めに、どのMODを作るにしても必須となるツールを紹介します。

1.1.QuickBMS
QuickBMSはファイルの変換ツールです。
専用のスクリプトを別途用意することで、.pakファイルを解凍することができます。
ツールは下記のサイトからダウンロードしてください。
http://aluigi.altervista.org/quickbms.htm

.pakの解凍にはFF7R専用のスクリプトが必要となります。OpenFF7Rで入手してください。

1.2.UE Viewer (umodel)
UE ViewerはUE4のアセット解析ツールです。
3Dモデルやテクスチャを編集可能なフォーマットへ変換できます。
ツールは下記のサイトからダウンロードしてください。
UE Viewer | Gildor's Homepage

ただし、実行ファイル(.exe)はFF7R専用にビルドされたものを使う必要があります。OpenFF7Rで入手してください。

1.3.UnrealPak
UnrealPakはアセットを.pakに変換するためのツールです。
ただし、Githubで公開されているUnrealPakはゲーム開発用のものであり、MOD作成には利用できないので注意してください。
例えば、Flaffy Quack氏によるMOD用のビルドが こちら からダウンロードできます。
ただし、かなり古いビルドなので圧縮形式が最新のものとは異ります。
以前はOpenFF7RでFF7R用のビルドを公開している人がいたのですが、現在は投稿が削除されてしまったため入手不可能となっています。

2.モデルMOD用ツール

テクスチャやメッシュの編集など、モデルMODの作成に必要なツールを紹介します。

2.1.DDS Injector
GitHub - SamsamTS/DDSInjector
Samsam氏が作成したテクスチャMOD用のツールです。
DDSファイルをテクスチャアセットに埋め込むことができます。
GUI操作のため使いやすいのですが、
「元のアセットと同じサイズのテクスチャしか埋め込めない」
「認識できないフォーマットがある」
など、デメリットもあります。

2.2.FF7R DDS Tools
GitHub - matyalatte/FF7R-DDS-tools
Samsam氏のツールのデメリットを全て解消したツールです。
任意サイズのDDSを埋め込むことができます。
ただし、GUIには対応していません。
僕が作りました。

2.3.pskプラグイン
GitHub - Befzz/blender3d_import_psk_psa
モデルデータをBlenderで読み込むためのプラグインです。

2.4.pskプラグイン(カスタマイズ版)
GitHub - matyalatte/blender3d_import_psk_psa
MOD用にカスタマイズされたpskプラグインです。
スケールの変更、スムーズシェーディングの適用、オブジェクト名の変更など、MODの作成に必要な手順を自動化します。
僕が作りました。

2.5.FF7R Mesh Importer
GitHub - matyalatte/FF7R-mesh-importer
UE4.18で作成したメッシュアセットをFF7Rのアセットに埋め込むことができます。
作成したアセットは.pakに変換することでMODとして利用できます。
僕が作りました。

2.6.3DMigoto
3DMigoto Base Mod for FFVII Remake at Final Fantasy VII Remake
動作中のゲームに対してメッシュデータを動的に書き換えるツールです。
専用のモデルデータを作成することでモデルMODを適用できるのですが、 MOD作成者だけでなくMOD利用者にも環境構築を強いる仕様となっており、敷居の高いツールとなっています。
モデルMODを作る場合は、なるべくFF7R mesh importerの利用を推奨します。

3.その他のツール

その他、MOD作成に使えるツールを紹介します。

3.1.FF7R Uasset Editor
GitHub - SamsamTS/FF7RUAssetEditor
Samsam氏が作成した、アセット内のnameリストを編集するためのツールです。
(nameリストとは、アセット内で参照されるクラス名やファイル名等の文字列のリストのことです。)

3.2.FF7R Text Mod Tools
GitHub - matyalatte/FF7R_text_mod_tools
字幕やメニュー画面等のテキストデータを編集するためのツールです。
僕が作りました。

3.3.FF7R Font Mod Tools
GitHub - matyalatte/FF7R-font-mod-tools
フォントを変更するためのツールです。
フォントファイル(.ttfまたは.otf)をゲームに取り込むことができます。
僕が作りました。

MODファイル(.pak)に関するTips

1. .pakが読み込まれる順序

ゲームを起動すると、"FFVIIRemakeIntergrade\End\Content\Paks"内に配置されている全ての.pakファイルが読み込まれます。
Paks内にフォルダがある場合は、そのフォルダ内に配置された.pakファイルも全て読み込まれます。
したがって、Paks内にMODファイルを入れるとMODが適用されるわけですが、

「.pakファイルは辞書順に読み込まれ、重複するデータは上書きされる。」

という仕様になっているため、注意が必要です。
つまり、

「本来のゲームデータより先にMODファイルが読み込まれると、MODデータが本来のゲームデータで上書きされるため、MODが適用されない。」

ということです。
"~mods"などのフォルダを作るのは、この「MODデータの上書き」を回避するためなのです。
(したがって、実はフォルダ名を"~hoge"や"zzz"にしても良い)
パッチファイルが"*_p.pak"という形式になっているのも同じ理由だと思われます。
Paks直下に.pakを配置する場合はファイル名に注意しましょう。

2. ファイル名の1文字目は"z"にする

前述した理由により、配布されている.pakファイルの多くはファイル名の1文字目が"z"になっています。
本来、"~mods"等のフォルダに.pakを配置するのであれば、ファイル名はなんでも良いわけですが、 いくら「~modsに配置しろ」とインストール手順に書いてあっても、必ず"Paks"直下に配置する人が現れます。
MODを作る際はファイル名の1文字目は"z"にしましょう。
(ただし、本来のゲームデータより後に読み込まれる名前であれば、1文字目を"z"にする必要はありません。)

表示言語の変え方

MODとは関係ありませんが、困っている人が（自分含め）何人かいたので書いておきます。

FF7Rの表示言語はアカウントの言語設定に依存しているため、ゲーム内で表示言語を変えることが出来ません。
表示言語を変えるためには、アカウントの言語設定を変える必要があります。
具体的な手順は以下の通りです。

  1. Epicストア右上のユーザーアイコンをクリックする
  2. 「アカウント」をクリックする

  3. ブラウザで設定画面が開いたら「個人情報」の「使用言語」を変更する

ちなみに、Epicストアの設定項目にある「言語」を変えても意味がないので注意してください。
FF7Rに反映されるのは、アカウント設定の「使用言語」という項目です。
